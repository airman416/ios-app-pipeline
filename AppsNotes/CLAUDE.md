# Claude Instructions for AppsNotes Vault

These notes are my working memory for a multi-session project. They are NOT the skill files — they are comprehensive records of everything the founder and I have discussed, so future sessions can continue without losing context.

## When to Write / Update Notes

- After every meaningful exchange where the founder gives input on a stage, framework, or decision
- When a new framework, book, or resource is introduced
- When a decision is made (even small ones — they add up)
- When I push back and the founder responds — capture both sides
- When open questions get answered — move them from "Open Questions" to the relevant section

Do NOT write notes for:
- Speculation or things not yet validated by the founder
- Implementation details that belong in the skill files themselves
- Anything already captured (update the existing note instead)

## How to Write Notes

Notes must be comprehensive and self-contained. A future Claude instance should be able to read any note cold and have full context — not just a summary, but the actual content, examples, nuance, and decisions. Do not summarize when you can quote. Do not paraphrase frameworks when you can state them precisely.

## When to Reference Notes

- At the START of every session — read 00-system-overview.md first, then any stage notes relevant to the current work
- Before asking the founder a clarifying question — check if it's already answered
- Before moving to a new stage — read that stage's note for open questions
- When something the founder says seems to contradict a prior decision — flag it and surface the conflict
